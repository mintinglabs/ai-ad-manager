# Lead Ads

## API Endpoints

### Read

```
GET /api/leads/forms?pageId=PAGE_ID
```
Returns all lead generation forms associated with a Facebook Page.

```
GET /api/leads/forms/:id/leads
```
Retrieves submitted lead data for a specific lead form.

```
GET /api/leads/ads/:adId
```
Returns leads generated by a particular ad.

### Create

```
POST /api/leads/forms
```

| Field | Type | Required | Notes |
|---|---|---|---|
| pageId | string | yes | Facebook Page ID |
| name | string | yes | Form display name |
| questions | array | yes | Array of question objects |
| privacy_policy_url | string | yes | Must be publicly accessible |

## Execution Workflow

Every write operation MUST follow this four-step pattern.

### Creating a Lead Form

**Step 1 READ** -- Fetch existing forms for the page.

```
GET /api/leads/forms?pageId=PAGE_ID
```

```metrics
Page: "My Business" (111222333)
Existing Lead Forms: 3
- "Contact Us Form" (12 leads)
- "Newsletter Signup" (47 leads)
- "Free Quote Request" (8 leads)
```

**Step 2 CONFIRM** -- Show the full form configuration.

```steps
Action: CREATE lead form
Page: "My Business" (111222333)
Name: "Spring Promotion - Lead Form"

Questions:
  1. EMAIL (pre-filled from profile)
  2. FULL_NAME (pre-filled from profile)
  3. PHONE_NUMBER (pre-filled from profile)
  4. Custom: "What product are you interested in?"
     Type: multiple choice
     Options: "Shoes", "Clothing", "Accessories"
  5. Custom: "Preferred contact method"
     Type: multiple choice
     Options: "Email", "Phone", "Text"

Privacy Policy: https://example.com/privacy ✓
```

**SAFETY CHECK**: Verify `privacy_policy_url` is present. If missing, stop and request it:
"A privacy policy URL is required by Facebook. Please provide the URL before I can create this form."

Ask: **"Should I proceed?"**

**Step 3 EXECUTE** -- Only after user confirms.

```
POST /api/leads/forms
```

**Step 4 VERIFY** -- Confirm the form was created.

```
GET /api/leads/forms?pageId=PAGE_ID
```

```metrics
Lead Form Created Successfully
ID: 456789...
Name: "Spring Promotion - Lead Form"
Questions: 5
Privacy Policy: ✓
```

```quickreplies
["Create ad with this form", "View existing leads", "Create another form", "Set up lead notification webhook"]
```

### Connecting a Form to an Ad

This is a multi-step process that spans skills:

1. Create the lead form (this skill).
2. Create a campaign with `OUTCOME_LEADS` objective (campaign-manager skill).
3. Create an ad set with `optimization_goal: LEAD_GENERATION` (adset-manager skill).
4. Create an ad creative referencing the form's `lead_gen_form_id` (creative-manager skill).
5. Create an ad linking the creative to the ad set (ad-manager skill).

Show each step in sequence, confirming before each execution.

## Safety Guardrails

- **Privacy policy required**: Every lead form MUST include `privacy_policy_url`. Block creation if missing and prompt user for the URL.
- **Form preview**: Always show the full question list and form structure in a `steps` block before creating. Users must see exactly what they are asking leads.
- **Deletions**: Require explicit confirmation. Warn that any ads using the form will stop collecting leads.
- **Objective match**: Verify the parent campaign uses `OUTCOME_LEADS` objective. Lead forms will not work with other objectives.
- **PII handling**: Lead data contains personal information. Remind users about data handling obligations when exporting.

## Quick Reference

### Question Types

**Pre-filled fields** (auto-populate from Facebook profile):

| Field | Key |
|---|---|
| Email | `EMAIL` |
| Full name | `FULL_NAME` |
| First name | `FIRST_NAME` |
| Last name | `LAST_NAME` |
| Phone | `PHONE_NUMBER` |
| City | `CITY` |
| State | `STATE` |
| Zip code | `ZIP_CODE` |
| Country | `COUNTRY` |
| Date of birth | `DATE_OF_BIRTH` |
| Gender | `GENDER` |
| Marital status | `MARITAL_STATUS` |
| Military status | `MILITARY_STATUS` |
| Work email | `WORK_EMAIL` |
| Job title | `JOB_TITLE` |
| Company name | `COMPANY_NAME` |

**Custom questions:**
- Free-text: user types an answer
- Multiple choice: user selects from predefined options
- Conditional: appears based on previous answers

### Lead Data Format

Each submission returns a `field_data` array:

```json
{
  "field_data": [
    { "name": "email", "values": ["user@example.com"] },
    { "name": "full_name", "values": ["Jane Doe"] },
    { "name": "custom_question_key", "values": ["Selected option"] }
  ]
}
```

`values` is always an array, even for single-value answers.

### Form Creation Example

```json
{
  "pageId": "111222333",
  "name": "Free Consultation Form",
  "questions": [
    { "type": "EMAIL" },
    { "type": "FULL_NAME" },
    { "type": "PHONE_NUMBER" },
    {
      "type": "CUSTOM",
      "key": "service_interest",
      "label": "Which service are you interested in?",
      "options": [
        { "value": "consulting", "key": "consulting" },
        { "value": "development", "key": "development" },
        { "value": "design", "key": "design" }
      ]
    }
  ],
  "privacy_policy_url": "https://example.com/privacy"
}
```
